02 — FEATURES

What You Can Do

Every feature listed below works out of the box with no extra configuration required.

⚡

Real-Time Collaboration

Multiple users can type simultaneously. Changes appear on every connected screen instantly with zero conflicts, powered by Yjs CRDT.

🖱️

Live Cursors

See every collaborator's cursor position in real time with a colored floating label showing their name. Uses the Awareness protocol.

👤

User Presence

Colored avatar bubbles in the top bar show who is currently in the document. Click your chip to change your display name anytime.

📝

Rich Text Editing

Full TipTap editor with Headings (H1/H2/H3), Bold, Italic, Underline, Inline Code, Code Block, Bullet Lists, and Ordered Lists.

💾

Auto-Save to PostgreSQL

The Yjs document state is automatically flushed to PostgreSQL as a binary BYTEA snapshot every 5 seconds and on disconnect.

🔄

Offline & Reconnect

If a user loses connection, y-websocket reconnects automatically. The CRDT merges all offline changes without conflicts on reconnect.

✏️

Inline Title Editing

Click the document title in the top nav to rename it. Saves to the database on blur or Enter. Press Escape to cancel.

📋

Document Management

Create, list, open, and delete documents from the Home dashboard. Hover any card to reveal the delete button.

🚀

Horizontal Scaling

Redis pub/sub propagates Yjs updates between multiple backend instances, making the system ready to run behind a load balancer.

04 — API REFERENCE

Endpoints

The backend exposes a REST API for document management and a WebSocket endpoint for real-time collaboration.

Method Endpoint Description Body / Notes
GET /api/documents List all documents Returns title, dates, active_users count
POST /api/documents Create new document { title: "My Doc" }
GET /api/documents/:id Get single document Returns metadata + active_users
PATCH /api/documents/:id Rename document { title: "New Name" }
DELETE /api/documents/:id Delete document Returns 204 No Content
GET /api/documents/:id/snapshots List version snapshots Returns last 20 saved snapshots
GET /health Server health check Returns DB status + server time
WS /ws/:documentId Real-time collab socket Binary y-websocket protocol (sync + awareness)
05 — SETUP & RUNNING

Getting Started

Two ways to run the project — Docker (recommended) for one-command startup, or manually for development.

1

Start PostgreSQL + Redis

Use Docker to spin up the infrastructure without installing anything locally.

docker-compose up postgres redis -d
2

Install & Run Backend

Install dependencies, run the database migration, then start the dev server on port 4000.

cd backend
npm install
npm run migrate
npm run dev
3

Install & Run Frontend

Open a second terminal, install dependencies, and start Vite on port 5173.

cd frontend
npm install
npm run dev
4

Open in Browser

Navigate to the app. Open in multiple tabs or browsers to test real-time collaboration.

http://localhost:5173
★

Alternative: Full Docker

Build and run everything (backend + frontend + DB + Redis) with a single command.

docker-compose up --build
